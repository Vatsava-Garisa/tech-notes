
When a GraphQL request hits your server:
1. Request hits `/graphql`.
2. Context is created.
3. Schema validates query.
4. Execution engine walks the query tree.
5. Resolvers execute field-by-field.
6. Response is constructed.

This is where GraphQL becomes fundamentally different from REST. 

### Example
```
query: {
	user(id: "2") {
		name
		posts {
			title
		}
	}
}
```
- Resolves `user`
- Then resolves `name`
- Then resolves `posts`
- Then for each post resolve `title`

GraphQL executes field-level resolution.
This leads to a major production problem. (N + 1 problem)
