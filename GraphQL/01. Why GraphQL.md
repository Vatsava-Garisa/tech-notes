
### REST Model
- Multiple endpoints.
- Over-fetching / under-fetching.
- Versioning complexity.
- Tight coupling to resource shape.
- HTTP status driven.
- Controller-based: `Route -> Controller -> Service -> DB`.
```
GET /users
GET /users/123
GET /users/123/comments
```
### GraphQL Model
- Single endpoint.
- Client specifies exactly what it needs. The query shape defines the response shape. The server resolves only requested fields.
- Strong schema evolution.
- Application level errors.
	- GraphQL communicates using HTTP.
	- Always returns 200 even with errors.
	- Usually errors are inside the `errors` array.
```
{
	"data": null,
	"errors": [
		{
			"message": "User not found",
			"extensions": {
				"code": "NOT_FOUND"
			}
		}
	]
}
```
- Resolver-based: `Schema -> Resolver -> Service -> DB`.
```
POST /graphql

query {
	user(id: "1") {
		name
		posts {
			title
		}
	}
}
```
