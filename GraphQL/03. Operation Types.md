
Three main operation types:
1. Query
2. Mutation
3. Subscription

### **Query:**
- Used for read-only operations.
- Must be side-effect free.
- Can be parallelized by GraphQL engine.
- **IMPORTANT:** Resolvers at the same level can run in parallel.
```
query {
	user(id: "1") {
		name
	}
}

/* Think: GET in REST. */
```
### **Mutation:**
- Used for write operations.
- Executed sequentially.
- Can modify state.
- **IMPORTANT**: Always executed sequentially to preserve consistency.
	- Deterministic state transitions.
	- Write consistency.
	- Predictable ordering.
	- Avoid race conditions inside the same request.
```
mutation {
	createUser(input: {...}) {
		id
	}
}

/* Think: POST / PUT / DELETE in REST */
```
### **Subscription:**
- Used for real-world updates.
- Uses WebSockets.
- Event-driven.
```
subscription {
	postCreated {
		id
		title
	}
}

/* Think: Pub/Sub system */
```
